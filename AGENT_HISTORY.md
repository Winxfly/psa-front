# PSA Frontend — История разработки

## Задача

Создать легковесный фронтенд для PSA (Professional Skills Analyzer) — сервиса анализа востребованных навыков на основе данных hh.ru API.

**Требования:**
- Без фреймворков (чистый HTML + CSS + JS)
- Только публичные API (административные не нужны)
- Минималистичный дизайн без анимаций
- Быстрая загрузка

**Функционал:**
1. Главная страница: список профессий кнопками + поиск
2. Страница профессии: информация + две таблицы навыков (формальные и извлечённые)

### Этап 3: JavaScript логика

**Реализовано:**
- Загрузка профессий: `GET /api/v1/professions`

- Живой поиск по названию профессии

- Рендеринг карточек и таблиц
- Обработка состояний (loading, error, empty)

**Структура кода:**
```

```

### Этап 4: Проблемы и решения

#### Проблема 1: CORS

**Симптомы:**
- Данные не загружаются, в консоли ошибка CORS
- Бэкенд отвечает 200 OK, но браузер блокирует ответ

**Причина:**
- Фронтенд на `http://localhost:3000`, бэкенд на `http://localhost:8080`
- CORS middleware на бэкенде не настроен на localhost:3000

**Решение:**
Пользователь добавил в `.env` бэкенда:
```bash
CORS_ALLOWED_ORIGINS=http://localhost:3000,http://localhost:8080
```

CORS middleware уже был реализован в бэкенде (файл `internal/handler/http/middleware/cors/cors.go`).

---

#### Проблема 2: Класс hidden не удаляется

**Симптомы:**
- В консоли видно: "Получено профессий: 35", "Рендеринг завершён, дочерних элементов: 35"
- На странице пусто

**Диагностика:**
- Добавил логирование в `renderProfessions()`
- Обнаружил: у элемента `professions-list` класс `hidden` остаётся
- В логе: `class="professions-grid hidden"`

**Причина:**
- Функция `showLoading()` добавляет `hidden` к `contentElement`
- После загрузки данных `renderProfessions()` не удаляет этот класс
- Дополнительно: `handleRoute()` вызывает `showProfessionsList()` до загрузки данных, что рендерит пустой массив и показывает `professions-empty`

**Решение:**
Добавил удаление класса `hidden` в `renderProfessions()`:
```javascript
elements.professionsList.classList.remove('hidden');
elements.professionsList.innerHTML = '';
```

Теперь класс удаляется перед рендерингом данных.

## История изменений

### 2026-03-16 — Рефакторинг layout графика

**Проблема:**
- График накладывался на заголовок "Динамика вакансий" и данные диапазона
- Использовалось `position: absolute` для `.chart-header`, что приводило к наложению элементов

**Решение:**
- Убрано абсолютное позиционирование у `.chart-header`
- Изменена структура на нормальный поток: заголовок → данные диапазона → график → контролы
- Данные диапазона вынесены в отдельный блок с фоном (`.chart-range-info`)
- Добавлен новый класс `.chart-header-top` для верхней строки с заголовком и индикатором
- Высота графика зафиксирована на 400px через обёртку canvas
- Когда ничего не выбрано — данные диапазона скрыты (класс `hidden`)
- При выборе двух точек данные отображаются в блоке с датой слева и изменением справа

**Изменённые файлы:**
- `css/main.css` — переработаны стили контейнера графика
- `js/pages/profession.js` — обновлена HTML-структура графика
- `js/pages/trends.js` — обновлена HTML-структура графика

**Fix: график не прыгает при выделении диапазона**

**Проблема:**
- При выделении диапазона блок с данными появлялся и сдвигал график вниз

**Решение:**
- Использован `visibility: hidden` вместо `display: none`
- Блок `.chart-range-info` занимает место всегда
- Добавлен класс `.empty` для скрытия контента
- При инициализации блок скрывается, но место сохраняется

**Изменённые файлы:**
- `css/main.css` — добавлен `visibility: hidden` и `.empty` класс
- `js/pages/profession.js` — обновление методов `_hideRangeInfo()`, `_showRangeInfo()`

**Refactor: данные диапазона в одну линию с заголовком**

**Проблема:**
- Отдельный блок с данными диапазона занимал много места

**Решение:**
- Данные диапазона перемещены в одну линию с заголовком "Динамика вакансий"
- Порядок: заголовок | данные диапазона | индикатор изменения
- Использован `visibility: hidden` для скрытия пустого состояния
- Все элементы центрированы по вертикали

**Изменённые файлы:**
- `css/main.css` — стили для `.chart-header-top` и `.chart-range-info`
- `js/pages/profession.js` — HTML-структура заголовка графика

**Fix: кнопки периода на подложке с графиком**

**Проблема:**
- Кнопки выбора периода (`.chart-controls`) перекрывали навыки
- Использовалось `position: absolute`, кнопки выпадали из потока

**Решение:**
- Убрано `position: absolute` у `.chart-controls`
- Кнопки теперь в нормальном потоке внутри `.chart-container`
- График и кнопки на одной подложке
- Ничего не перекрывается

**Изменённые файлы:**
- `css/main.css` — удалено абсолютное позиционирование у `.chart-controls`

**Feat: цвет выделения диапазона на графике**

**Изменения:**
- Зелёный (`#9ece6a` с прозрачностью 0.1) если изменение положительное
- Красный (`#f7768e` с прозрачностью 0.1) если изменение отрицательное
- Цвета соответствуют стилям `.positive` и `.negative`

**Изменённые файлы:**
- `js/pages/profession.js` — обновление `_createRangeHighlightPlugin()`
